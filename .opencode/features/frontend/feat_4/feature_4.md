# Feature 4: Страница регистрации пользователя

## Описание задачи

Создать полноценную страницу регистрации пользователя с интеграцией в систему аутентификации.

## Контекст

Проект использует:
- React 18+ с TypeScript
- React Router 6 для роутинга
- Material UI 7 для компонентов
- Redux Toolkit + RTK Query для управления состоянием
- Jest + React Testing Library для unit/integration тестов
- Playwright для E2E тестирования
- Feature-Sliced Design (FSD) архитектура

Существующие компоненты:
- LoginPage - страница входа
- ProtectedRoute - защита маршрутов
- useRegister hook - логика регистрации
- authApi - RTK Query endpoints

## Декомпозиция

### Sub-task 1: UI компонент RegistrationPage + unit тесты
- Создать RegistrationPage компонент
- Добавить форму с полями: email, password, confirm password, role
- Добавить валидацию полей
- Написать unit тесты

### Sub-task 2: Роутинг + навигация
- Добавить маршрут /register в router
- Добавить ссылку на регистрацию со страницы логирования
- Добавить редирект на /login после успешной регистрации

### Sub-task 3: Интеграция с useRegister + интеграционные тесты
- Интегрировать RegistrationPage с useRegister hook
- Обработать успешную регистрацию (редирект на login)
- Обработать ошибки регистрации
- Написать интеграционные тесты

### Sub-task 4: E2E тесты регистрации
- Создать E2E тест: tests/e2e/auth/register.spec.ts
- Тест: отображение формы регистрации
- Тест: валидация при пустых полях
- Тест: валидация при несовпадении паролей
- Тест: успешная регистрация -> редирект на login
- Тест: полный цикл регистрации + логина -> home

## Зависимости

- React Router 6 (существующий)
- MUI 7 (существующий)
- useRegister hook (существующий)
- authApi (существующий)

## Оценка сложности

Средняя (3/5)

## Критерии приёмки

1. Страница регистрации отображается по маршруту /register
2. Форма содержит поля: email, password, confirm password, role
3. Валидация работает корректно
4. При успешной регистрации - редирект на /login
5. Со страницы логирования есть ссылка на регистрацию
6. Защищенные маршруты недоступны без авторизации
7. Все unit/integration тесты проходят
8. E2E тесты проходят
