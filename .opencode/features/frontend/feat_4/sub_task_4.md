# Sub-task 4: E2E тесты регистрации

## Описание задачи

Создать E2E тесты для полного цикла регистрации пользователя.

## Способ решения

Использовать Playwright:
- tests/e2e/auth/register.spec.ts
- Использовать те же паттерны, что и в login.spec.ts

## Подготовка тесткейсов для TDD

### E2E тесты (до реализации - должны упасть)

1. **display_registration_form** - Отображение формы регистрации
   - Ожидание: отображается заголовок "Регистрация"
   - Ожидание: отображаются поля email, пароль, подтверждение пароля, роль
   - Ожидание: отображается кнопка "Зарегистрироваться"

2. **validation_empty_submit** - Валидация при пустой отправке
   - Ожидание: показываются ошибки валидации

3. **validation_password_mismatch** - Валидация несовпадения паролей
   - Ожидание: показывается ошибка "Пароли не совпадают"

4. **navigation_to_login** - Навигация на страницу логирования
   - Ожидание: при клике на "Войти" - переход на /login

5. **full_registration_flow** - Полный цикл регистрации
   - Действие: заполнить форму регистрации
   - Действие: отправить форму
   - Ожидание: редирект на /login

6. **login_after_registration** - Логин после регистрации -> home
   - Действие: зарегистрировать пользователя
   - Действие: войти с теми же учетными данными
   - Ожидание: редирект на / (home)

7. **protected_route_requires_auth** - Защищенный маршрут требует авторизации
   - Действие: перейти на / без авторизации
   - Ожидание: редирект на /login

## Ожидаемый результат

- Создан: frontend/tests/e2e/auth/register.spec.ts
- Все E2E тесты проходят

## Критерии приёмки

1. E2E тесты проходят
2. Полный цикл: регистрация -> логин -> home работает
3. Protected route работает корректно
