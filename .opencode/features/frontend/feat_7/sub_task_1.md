# Sub-task 1: Добавить API эндпоинт для получения цены монеты

## Описание

Добавить публичный метод в assets.service.ts для получения текущей цены монеты по символу через CoinMarketCap API. Этот метод будет использоваться при создании токена, если текущая цена не передана.

## Способ решения

Использовать существующий код из asset-update.service.ts (метод fetchFromCoinMarketCap) как референс. Создать публичный метод в AssetsService, который:
1. Принимает символ монеты (например, "BTC")
2. Делает запрос к CoinMarketCap API
3. Возвращает текущую цену или null

## Подготовка тесткейсов для TDD

1. **Тест: Успешное получение цены**
   - Вход: symbol = "BTC"
   - Ожидание: возвращает число (текущая цена BTC)

2. **Тест: Монета не найдена**
   - Вход: symbol = "INVALID"
   - Ожидание: возвращает null

3. **Тест: API ключ не установлен**
   - Вход: symbol = "ETH", нет API ключа
   - Ожидание: возвращает null, логируется предупреждение

## Ожидаемый результат

Метод `getCryptoPrice(symbol: string): Promise<number | null>` добавлен в AssetsService.

## Критерии приёмки

- [ ] Метод возвращает цену при успешном API ответе
- [ ] Метод возвращает null при ошибке
- [ ] Unit тесты покрывают метод
