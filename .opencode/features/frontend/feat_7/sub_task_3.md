# Sub-task 3: Добавить эндпоинт для обновления всех активов пользователя

## Описание

Добавить POST /assets/refresh эндпоинт в assets.controller.ts, который позволяет пользователю вручную обновить цены всех своих активов через CoinMarketCap API. Это нужно, так как автоматическое обновление через cron не обновляет UI.

## Способ решения

1. Добавить метод refreshAll в AssetsController
2. Использовать существующий AssetUpdateService для обновления
3. Вернуть список обновлённых активов

## Подготовка тесткейсов для TDD

1. **Тест: Успешное обновление**
   - Вход: POST /assets/refresh (пользователь с активами)
   - Ожидание: возвращается массив обновлённых активов

2. **Тест: Нет активов для обновления**
   - Вход: POST /assets/refresh (пользователь без активов)
   - Ожидание: возвращается пустой массив

## Ожидаемый результат

Эндпоинт POST /assets/refresh добавлен и работает с JWT авторизацией.

## Критерии приёмки

- [ ] Эндпоинт защищён JWT Guard
- [ ] Возвращает массив обновлённых активов
- [ ] Обрабатывает случай, когда нет активов
- [ ] Интеграционный тест покрывает эндпоинт
