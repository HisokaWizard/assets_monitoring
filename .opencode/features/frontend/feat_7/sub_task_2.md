# Sub-task 2: Обновить создание актива с автоматическим получением цены

## Описание

Модифицировать метод create в assets.service.ts для автоматического получения текущей цены, если она не передана в CreateAssetDto. Это позволит пользователю создавать токены только с обязательными полями (тикер, количество, средняя цена).

## Способ решения

1. В методе create проверить, передан ли currentPrice
2. Если не передан и тип актива 'crypto', вызвать getCryptoPrice (из sub-task 1)
3. Если цена получена, использовать её для расчёта полей multiple, totalChange
4. Сохранить актив с рассчитанными полями

## Подготовка тесткейсов для TDD

1. **Тест: Создание с указанной ценой**
   - Вход: { type: 'crypto', symbol: 'BTC', amount: 1, middlePrice: 50000, currentPrice: 55000 }
   - Ожидание: актив создаётся с currentPrice = 55000

2. **Тест: Создание без цены - цена получена**
   - Вход: { type: 'crypto', symbol: 'BTC', amount: 1, middlePrice: 50000 }
   - Ожидание: актив создаётся с текущей ценой из API

3. **Тест: Создание без цены - API недоступен**
   - Вход: { type: 'crypto', symbol: 'BTC', amount: 1, middlePrice: 50000 }
   - Ожидание: актив создаётся с currentPrice = 0

## Ожидаемый результат

Метод create в AssetsService автоматически получает цену, если она не передана.

## Критерии приёмки

- [ ] Актив создаётся без указания currentPrice
- [ ] При отсутствии цены используется 0
- [ ] Рассчитываются поля multiple и totalChange
- [ ] Unit тесты покрывают логику
