## 1. Что такое Rivalz и zNode — коротко

**Rivalz** — это децентрализованная сеть (DePIN), которая обеспечивает инфраструктуру для работы ИИ-агентов. Сеть работает на Base (L2 поверх Ethereum).

**zNode** — это нода-валидатор, которая проверяет целостность данных в сети Rivalz. Ваша нода будет подтверждать, что данные, хранящиеся у других участников сети, не были изменены или повреждены. За эту работу вы получаете вознаграждение в токенах $RIZ.

Чтобы запустить zNode, необходимо иметь **лицензию (NFT)** — она подтверждает ваше право быть валидатором.

## 2. Что вам понадобится

### Оборудование

| Компонент | Минимум | У вас |
|-----------|---------|-------|
| Устройство | Raspberry Pi 4 | ✅ Raspberry Pi 4 |
| RAM | 2 ГБ (рекомендовано 8 ГБ) | ✅ 8 ГБ |
| Хранилище | 128 ГБ свободного места | SD-карта / SSD (см. ниже) |
| Интернет | Стабильное соединение | Ethernet рекомендуется |
| ОС | Ubuntu 20.04+ / Raspberry Pi OS 64-bit | ✅ Raspberry Pi OS 64-bit |

### Программное обеспечение

- Node.js версии 20.x или выше
- npm (идёт в комплекте с Node.js)

### Крипто-активы

- Лицензия zNode (NFT) на вашем кошельке
- Немного ETH в сети **Base** для оплаты газа при делегировании
- Криптокошелёк (MetaMask или аналог) с подключённой сетью Base

### Рекомендация по хранилищу

Raspberry Pi работает быстрее и стабильнее с **USB SSD**, чем с SD-картой. Если планируете держать ноду долго — подключите SSD через USB 3.0 и загружайте систему с него.

## 3. Подготовка Raspberry Pi

### 3.1. Подключение к Raspberry Pi

Если у вас монитор и клавиатура подключены к Pi — просто откройте терминал (значок `>_` на панели задач).

Если вы работаете удалённо через SSH с другого компьютера:

```bash
ssh pi@<IP-адрес-вашего-Pi>
```

Пароль по умолчанию: `raspberry` (если не меняли при установке ОС). **Обязательно смените его** после первого входа:

```bash
passwd
```

> **Как узнать IP-адрес Pi:** на самом Pi выполните `hostname -I`, либо посмотрите в настройках роутера.

### 3.2. Обновление системы

Первым делом обновим все пакеты, чтобы система была актуальной:

```bash
sudo apt update && sudo apt upgrade -y
```

Это может занять несколько минут. Дождитесь завершения.

### 3.3. Установка необходимых утилит

Установим инструменты, которые понадобятся далее:

```bash
sudo apt install -y curl wget git screen
```

Что это:
- `curl` и `wget` — скачивание файлов из интернета
- `git` — управление версиями (может понадобиться)
- `screen` — позволяет держать ноду запущенной в фоне

### 3.4. Проверка архитектуры

Убедимся, что система 64-битная (это важно для Node.js):

```bash
uname -m
```

Ожидаемый результат: **`aarch64`**. Если вы видите `armv7l` — у вас 32-битная ОС, и нужно переустановить Raspberry Pi OS 64-bit.

## 4. Установка Node.js

zNode требует Node.js версии 20 или выше. Стандартные репозитории Raspberry Pi OS содержат старую версию, поэтому мы установим Node.js через **nvm** (Node Version Manager) — это самый надёжный способ на ARM-устройствах.

### 4.1. Установка nvm

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash
```

После установки нужно перезагрузить профиль терминала:

```bash
source ~/.bashrc
```

### 4.2. Установка Node.js 20

```bash
nvm install 20
```

Установка может занять 1–2 минуты. nvm автоматически скачает версию для ARM64.

### 4.3. Проверка установки

```bash
node -v
```

Вы должны увидеть что-то вроде `v20.x.x` (например, `v20.18.1`).

```bash
npm -v
```

Должна вывестись версия npm (например, `10.x.x`).

Если обе команды выдают версии — всё установлено верно.

## 5. Установка zNode CLI

### 5.1. Глобальная установка пакета

```bash
npm install -g rivalz-znode-cli
```

Это установит CLI-утилиту `znode` глобально. Процесс может занять 2–5 минут на Raspberry Pi.

> **Если появляется ошибка прав доступа (EACCES):** это нормально при использовании nvm — пакеты устанавливаются в домашнюю директорию, ошибки быть не должно. Но если она есть, выполните:
> ```bash
> mkdir -p ~/.npm-global
> npm config set prefix '~/.npm-global'
> echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
> source ~/.bashrc
> npm install -g rivalz-znode-cli
> ```

### 5.2. Проверка установки

```bash
znode -V
```

Должна отобразиться версия zNode (например, `1.x.x`). Если команда найдена и выводит версию — установка прошла успешно.

## 6. Первый запуск zNode

### 6.1. Запуск ноды

```bash
znode run
```

При **первом** запуске CLI задаст вам несколько вопросов:

1. **Введите приватный ключ вашего кошелька** — это ключ от кошелька, на котором находится лицензия NFT.

> ⚠️ **ВНИМАНИЕ ПО БЕЗОПАСНОСТИ:**  
> Приватный ключ — это полный доступ ко всем средствам на вашем кошельке. Рекомендуем:
> - Использовать **отдельный кошелёк** специально для ноды
> - Перевести на него только лицензию NFT и немного ETH на газ
> - **Никогда** не вводите приватный ключ от основного кошелька с крупными суммами

2. **Укажите размер диска**, который вы готовы выделить для ноды (в ГБ).

После ввода данных нода начнёт работу. Вы увидите лог с информацией о подключении к сети.

### 6.2. Получение Node ID

Остановите ноду (`Ctrl + C`) и выполните:

```bash
znode info
```

Эта команда покажет информацию о вашей ноде. **Скопируйте Node ID** — он выглядит как `0x9f7c...33c` (длинный адрес, начинающийся с `0x`). Он понадобится на следующем шаге.

Также можно получить Node ID отдельной командой:

```bash
znode show-node-id
```

## 7. Делегирование лицензии через браузер

Делегирование — это привязка вашей лицензии (NFT) к конкретной ноде. Без этого шага нода не будет получать задания и награды.

### 7.1. Подготовка

Убедитесь, что:
- На вашем кошельке есть **как минимум 1 NFT-лицензия** zNode
- На кошельке есть **немного ETH в сети Base** для оплаты газа (обычно меньше $1)
- У вас **скопирован Node ID** из предыдущего шага

### 7.2. Открытие панели управления

На **любом** компьютере (или на самом Pi, если у вас подключён монитор) откройте браузер и перейдите по адресу:

```
https://znode.rivalz.ai/licenses
```

### 7.3. Подключение кошелька

1. Нажмите кнопку **"Connect Wallet"** в правом верхнем углу
2. Выберите ваш кошелёк (MetaMask, WalletConnect и т.д.)
3. Подтвердите подключение в кошельке
4. Убедитесь, что кошелёк переключён на **сеть Base**

### 7.4. Делегирование лицензии

1. В поле **"Node ID"** вставьте скопированный Node ID вашей ноды
2. В поле **"Number of licenses to delegate"** укажите количество лицензий, которые хотите привязать к этой ноде (обычно `1`)
3. Нажмите кнопку **"Delegate"**
4. В кошельке появится запрос на подтверждение транзакции — **подтвердите** её
5. Дождитесь, пока транзакция будет обработана сетью (обычно 10–30 секунд)
6. Если всё прошло успешно, страница автоматически обновится и покажет вашу делегированную лицензию

---

## 8. Запуск ноды после делегирования

Вернитесь в терминал Raspberry Pi и запустите ноду снова:

```bash
znode run
```

Теперь нода увидит делегированную лицензию и начнёт полноценную работу. В логах вы должны увидеть информацию о привязанных лицензиях.

## 9. Фоновый режим работы (screen)

Если вы закроете терминал или разорвёте SSH-соединение — нода остановится. Чтобы этого не произошло, используйте `screen`.

### 9.1. Создание сессии screen

```bash
screen -S znode
```

Это создаст именованную сессию. Теперь запустите ноду внутри неё:

```bash
znode run
```

### 9.2. Отключение от сессии (нода продолжит работать)

Нажмите последовательно:

```
Ctrl + A, затем D
```

Вы вернётесь в основной терминал, а нода продолжит работать в фоне.

### 9.3. Возврат к сессии (посмотреть логи)

```bash
screen -r znode
```

### 9.4. Список активных сессий

```bash
screen -ls
```

> **Совет:** после отключения от screen можно спокойно закрывать SSH — нода будет работать, пока Pi включена.

## 10. Проверка работы ноды

### 10.1. Через терминал

Подключитесь к сессии screen и проверьте логи:

```bash
screen -r znode
```

Нода должна выводить периодические сообщения о своей работе. Ошибок подключения быть не должно.

Также выполните:

```bash
znode info
```

Эта команда покажет:
- Ваш Node ID
- Количество делегированных лицензий
- Текущий статус ноды

### 10.2. Через браузер (панель управления)

1. Откройте [https://znode.rivalz.ai](https://znode.rivalz.ai) в браузере
2. Подключите кошелёк
3. Перейдите в раздел **Licenses**
4. Вы должны увидеть свою лицензию со статусом делегирования и привязанным Node ID
5. Если нода онлайн и работает — статус будет отображаться как активный

### 10.3. Быстрая проверка (чек-лист)

- [ ] Команда `znode -V` возвращает версию
- [ ] Команда `znode info` показывает ваш Node ID и лицензию
- [ ] На сайте `znode.rivalz.ai` лицензия привязана к вашему Node ID
- [ ] Нода работает в screen-сессии и выводит логи без ошибок
- [ ] SSH-соединение можно разорвать без остановки ноды

## 11. Обновление zNode

Разработчики Rivalz периодически выпускают обновления. Обновляйтесь регулярно.

### Способ 1: Через встроенную команду

```bash
znode update-version
```

### Способ 2: Через npm (если Способ 1 не работает)

```bash
npm update -g rivalz-znode-cli
```

После обновления перезапустите ноду:

```bash
# Подключитесь к screen-сессии
screen -r znode

# Остановите ноду: Ctrl + C

# Запустите заново
znode run

# Отключитесь от screen: Ctrl + A, затем D
```

## 12. Автозапуск при перезагрузке (systemd)

Чтобы нода автоматически запускалась после перезагрузки Raspberry Pi (например, после отключения электричества), создадим systemd-сервис.

### 12.1. Определение путей

Сначала узнаем, где находятся node и znode:

```bash
which node
which znode
```

Запомните эти пути (например, `/home/pi/.nvm/versions/node/v20.18.1/bin/node` и `/home/pi/.nvm/versions/node/v20.18.1/bin/znode`).

### 12.2. Создание файла сервиса

Замените пути в команде ниже на те, что получили на предыдущем шаге:

```bash
sudo nano /etc/systemd/system/znode.service
```

Вставьте следующее содержимое (отредактируйте пути и имя пользователя):

```ini
[Unit]
Description=Rivalz zNode Validator
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=pi
Environment=PATH=/home/pi/.nvm/versions/node/v20.18.1/bin:/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin
ExecStart=/home/pi/.nvm/versions/node/v20.18.1/bin/znode run
Restart=on-failure
RestartSec=30
StandardOutput=journal
StandardError=journal

[Install]
WantedBy=multi-user.target
```

> **Важно:** замените `/home/pi/.nvm/versions/node/v20.18.1/bin` на ваш реальный путь из `which node`. Замените `User=pi` на ваше имя пользователя, если оно отличается.

Сохраните файл: `Ctrl + O`, `Enter`, `Ctrl + X`.

### 12.3. Активация сервиса

```bash
sudo systemctl daemon-reload
sudo systemctl enable znode.service
sudo systemctl start znode.service
```

### 12.4. Проверка статуса

```bash
sudo systemctl status znode.service
```

Вы должны увидеть `active (running)`.

### 12.5. Просмотр логов сервиса

```bash
journalctl -u znode.service -f
```

Флаг `-f` показывает логи в реальном времени. Выход — `Ctrl + C`.

> **Если вы используете systemd — screen больше не нужен.** Сервис автоматически перезапустит ноду при сбоях и после перезагрузки.

## 13. Устранение неполадок

### Проблема: `znode: command not found`

Node.js и npm установлены через nvm, но путь не подхвачен. Решение:

```bash
source ~/.bashrc
```

Или добавьте путь вручную:

```bash
export NVM_DIR="$HOME/.nvm"
[ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh"
```

### Проблема: Ошибки при `npm install -g rivalz-znode-cli`

Если видите ошибки компиляции нативных модулей — установите инструменты сборки:

```bash
sudo apt install -y build-essential python3
```

Затем повторите установку:

```bash
npm install -g rivalz-znode-cli
```

### Проблема: ARM-совместимость (маловероятно, но возможно)

Raspberry Pi 4 работает на процессоре ARM (Cortex-A72), тогда как официальная документация рекомендует Intel Core i3. Пакет `rivalz-znode-cli` написан на Node.js и в большинстве случаев прекрасно работает на ARM64. Однако если внутри есть нативные бинарные зависимости, скомпилированные только под x86 — может понадобиться VPS вместо Pi.

Признаки проблемы:
- Ошибки вида `Error: ... is not a supported platform`
- Ошибки вида `exec format error`
- Segmentation fault при запуске

Если столкнулись с этим — обратитесь в Discord Rivalz для уточнения ARM-поддержки.

### Проблема: Нода запускается, но не получает задания

1. Убедитесь, что лицензия делегирована: проверьте на `https://znode.rivalz.ai/licenses`
2. Проверьте, что кошелёк содержит достаточно ETH в сети Base для газа
3. Перезапустите ноду: `Ctrl + C`, затем `znode run`

### Проблема: Потеря SSH-соединения — нода остановилась

Вы запустили ноду не через screen или systemd. Используйте один из этих способов (см. разделы 9 и 12).

### Проблема: Raspberry Pi перегревается

При длительной работе Pi может нагреваться. Рекомендации:
- Используйте радиаторы на процессоре
- Установите активное охлаждение (вентилятор)
- Убедитесь, что Pi стоит в проветриваемом месте
- Проверить температуру: `vcgencmd measure_temp`

Нормальная температура: до 70°C. Выше 80°C — нужно улучшить охлаждение.
